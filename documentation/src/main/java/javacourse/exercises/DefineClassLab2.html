This exercise is to try out some more features of defining classes:

  • interfaces
  • packages
  • initialising arrays
  • instanceof operator
  • abstract classes

To see your Sales classes in a Swing GUI, change your classes to implement the following interfaces:

  • SalesUnit should implement jarden.gui.SalesUnitIF
  • SalesPerson should implement jarden.gui.SalesPersonIF
  • SalesTeam should implement jarden.gui.SalesTeamIF

Now, in your SalesTest class, pass your top-level team to the constructor of the jarden.gui.SalesSwing class.

This illustrates one of the functions of interfaces: we can plug different classes together, without knowing what they are called or how they are coded, so long as they implement the agreed interfaces.

Solution in package sales

The theme chosen for the rest of this exercise is chess, using a very simple implementation of some of the pieces and their moves. If the thought of writing classes for chess fills you with horror, just use the classes in package solution.chess, and in a different package define a class with a main() method; most of the learning points are contained in this main() method.

Define an abstract class ChessPiece:

  • name - a string, e.g. "Rook"
  • white - a boolean
  • xPos, yPos - integers, with values from 1 to 8
  • a constructor
  • toString() - e.g. "White Rook(1, 1)"
  • move(x, y) - call validateMove(), and update xPos and yPos
  • validateMove(x, y) - abstract method
  • getName()
  • getShortName() - a String; defaults to 1st char of name

Define a class Rook as subclass of ChessPiece; overwrite method validateMove(x, y) to check that it is a valid move for a rook - either x or y must stay the same.

Create a Rook instance; print it; move it; print it again

Similarly define a class King, which can only move 1 square in any direction. Hover for hint

Now put ChessPiece, Rook and King into package chess, and put the main method into a class Main, in another package. See what happens with and without the import statements.

Define a method which accepts an array of ChessPieces and prints them. Keep a check on the number of kings, using instanceof operator. At the end of printing the pieces, print the number of kings.

Within main(), define an array of ChessPieces, and initialise it with different types of pieces. Pass this array to your new method to print the pieces and count the kings.

Solution: solution.ClassDefinition2 and classes in package solution.chess

You can try your classes with a Swing graphical user interface that we have provided. To use this:

  • to make sure your classes are properly defined, specify that your ChessPiece class implements interface jarden.gui.Chess
  • in your main method, create an array of Chess, containing an object of each Chess class you have defined.
  • create an object of type jarden.gui.ChessSwing, passing your chess array to the constructor, e.g.
    public static void main(String[] args) {
    Chess[] chessPieces = {
        new King(true, 5, 1),
        new Rook(true, 1, 1),
        new Knight(true, 2, 1)
    };
    new ChessSwing(chessPieces);
}
Now you should see your Chess pieces in action!
