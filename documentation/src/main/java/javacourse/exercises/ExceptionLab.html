Exceptions

Modify your method which prints a compass direction - see Methods part 2, or solution.Methods if you didn't complete this exercise. In the method, throw an IllegalArgumentException if the value passed is invalid. At this point don't try to catch the exception. Test the code with some valid chars, then an invalid one. Run your code to confirm an exception is thrown.

Now define a new exception called NoDirectionHomeException, which is a subclass of Exception. Throw NoDirectionHomeException instead of IllegalArgumentException. The compiler should now tell you that you have to declare this exception in the called method. Once you do that, the compiler should say you need to handle the exception in the calling method. Wrap the call in a try-catch block. Run your code to confirm that an exception is thrown and caught.

Assertions

Add an extra call to your method, with an invalid value, and assert that an exception is thrown. To do this:

  • wrap the invalid call in a try-catch block
  • after the call, add assert false, i.e. fail if the code reaches this point
  • in the catch block, report that the exception was thrown as expected.

Check the assert false is not reached. Now, to prove your code would pick up a programming error, introduce one by passing a valid direction and expecting it to fail. Don't forget to enable assertions!
