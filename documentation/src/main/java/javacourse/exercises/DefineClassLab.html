Define Product Class - team exercise

The instructor will demonstrate how to define a class, but with the help of all the students. Please feel free to contribute, and correct his 'deliberate' errors.

Define a class to hold the part number, description and price of a product. Provide a constructor, a method to return the variables concatenated into a String, and a method to return the price of the product. Within main(), invoke the product methods and print out the returned values.

Sample Solution: solution.Product

Define SalesUnit Class

The exercises from this and the following two chapters all develop a project to keep track of sales figures for sales people reporting to sales teams. At each stage, build on your solution to the previous exercise, or if you are falling behind, use the sample solution from the previous exercise.

  1. Create a file SalesUnit.java, containing public class SalesUnit, which defines someone or something (i.e. a sales person or a sales team) that sells products, hopefully for a profit. SalesUnit should contain the following members.
    SalesUnit
    String name
    double salesTarget
    SalesUnit(String name, double salesTarget)
    String getName()
    double getSalesTarget()
    String toString()

    getName() returns the name of the SalesUnit
    getSalesTarget() (hopefully you get the idea by now)
    toString() returns a String made from concatenating the name and salesTarget

    Create a file called SalesTest.java, containing a public class SalesTest, which in turn has a method main(). In main(), create several SalesUnit objects, and invoke the methods for these objects.

    Also in main(), test out the effects of using the modifiers private and public on variables and methods.

  2. Further develop the SalesUnit class:
    • check that the salesTarget is not less than 1000
    • hold the date the object was created; today's date can be obtained using something like:
      java.util.Date today = new java.util.Date();

Sample solution in package sales1
