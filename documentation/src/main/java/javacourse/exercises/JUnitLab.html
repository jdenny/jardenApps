JUnit Testing

We're going to write a test method for an algorithm to sort an array of integers. Then we're going to write the algorithm, and test if it works. Yes, we're writing the test first - which should fail - then we'll write the sort code.

Create a new package (e.g. myjunit). In that package create class MySort, containing skeletal versions of the sort methods:

public static void sort(int[] numbers)
public static void reverseSort(int[] numbers)
The methods are static because they don't use any instance data. For now the methods do nothing.

Create new JUnit Test Case:

  • right click on your package, New, New JUnit Test Case
  • call the class SortTest
  • Class under test (of course) is MySort
In the generated source, notice Eclipse has done some of the housekeeping for us.

Write a before method to create an integer array, as an instance variable.

Code the 2 test methods to call MySort, then assert that the values are as expected. For simplicity, only test 2 or 3 values in the array.

Run the JUnit tests, and confirm that they fail.

Now implement the 2 sort methods, until they pass the test.

Sample Solution: package solution.junit
