Constructors - Hotel

Probably 2 constructors for each type of booking:
  • no parameters, in case the user interface creates the booking then adds the data following further user interactions
  • all data, in case the user interface captures the information, then creates the booking.
Examples in Java:
public HotelBooking()
public HotelBooking(Date startDate, int personCount,
                    Date endDate, RoomType roomType)

Examples in Python:
def __init__(self, startDate = None, personCount = 0, endDate=None, roomType=None)
Note: Python and Java have different approaches to varying the parameters passed to a method: Java uses method overloading: multiple methods with the same name but different types of parameters; Python uses default parameter values.

Constructors - Quiz

For the Quiz types that generate questions - e.g. Arithmetic, Algebra - no parameters. All the variables used by the superclass Quiz - e.g. level, attempts - can be initialised with default values.

For Quiz types based on pre-defined questions, a parameter containing the data - a collection of question and answer pairs - or the location of the data, e.g. fileName or database table.

Constructor Advantages

Constructors ensure that any required initialisation is performed before any other methods are invoked. Would you want to book into a hotel before it was constructed?

Objects

Hotel. One object for each hotel booking.

Quiz. Two alternatives:

  • Create object each time user selects new quiz type
  • Have a caching mechanism; if Quiz object already in the cache, use this, otherwise create it. This requires a reset() method to put the state back to the beginning. Question: What should reset() do?
