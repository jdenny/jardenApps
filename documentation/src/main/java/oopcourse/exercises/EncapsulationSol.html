All variables should be encapsulated!

Encapsulation provides 2 main advantages:

  • the methods that give access to the data can enforce validation rules, e.g. speed of plane cannot be negative; only authorised people can take money out of your bank account
  • the implementation can change over time - the way the data is held and the rules for setting the data - without changing the program interface; this means we can change the implementation without changing the code that is using our class

Some languages, e.g. Java, C++, have another level of visibility: protected. Protected means the attribute is visible to subclasses. Some people think this is a hack, a quick fix which comes back to bite you later.

Constant values (static final in Java) are often public. Examples of constants in Quiz are:

  • prompts - 'What is the capital of {}?'

    default maximum value of operands in arithmetic questions
