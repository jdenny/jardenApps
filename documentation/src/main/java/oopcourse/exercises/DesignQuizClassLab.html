Redesign an existing application in an Object-Oriented way

Two programs have been written so far, and the outline of their current logic is as follows:

  • arithmetic quiz
    loop until user quits:
	get 2 random numbers, a, b (from 1 to 10)
	get random operator: +, -, *, /
	prompt for user input: what is <a> <op> <b>?
	test input against correct answer
	if wrong, print correct answer
  • text-based quiz
    define list of question/answer pairs (hard-coded in program)
loop until end of list or user quits:
	prompt user with next question
	test input against correct answer
	if wrong, print correct answer

The analysts have identified the following requirements to extend the application:

  • user interface is separate from business logic (so we can plug in different user interfaces: command-line, windows, web browser)
  • easy to plug in different quiz types; generated: arithmetic, algebra, series, word anagrams; pre-defined questions/answers, read from file or database
  • give the user 3 attempts to get the right answer; if fails after 3 attempts, reveal the answer
  • for generated questions (e.g. maths):
    • if user gets 3 consecutive questions right on first attempt then increase level of difficulty
  • for pre-defined questions (e.g. general knowledge):
    • if doesn't get it right first time, ask the same question again later

Exercise - specify classes

Your mission (should you choose to accept it) is to decide what classes are required. Then, for each class, what attributes - data and functions - are required.

What inheritance structure is appropriate? What factors determine inheritance?
