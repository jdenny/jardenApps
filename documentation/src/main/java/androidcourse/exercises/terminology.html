Android Terminology

Back to Exercise List

Action BarArea at the top of each screen to expose important actions and navigations, e.g. New, Search.
ActivityAndroid class that manages a single user screen
ADBAndroid Debug Bridge; terminal emulator: adb shell
ADTAndroid Development Tools: plug-in for Eclipse
APKApplication PacKage, an archive file of the complete application, with an .apk suffix; it contains code, data and resources; similar to Java .jar file (same format as .zip)
Application Componentbasic building block of an app; entry point for app; 4 types of component: Activity, Service, Content Provider, Broadcast Receiver
ARMProcessor architecture from ARM Holdings, used in most smartphones.
ARTAndroid Runtime; In Lollipop, the Dalvik Virtual Machine is completely replaced by ART. ART has many advantages over Dalvik VM such as AOT (Ahead Of Time) compilation and improved garbage collection which boost the performance of apps significantly.
Broadcast Receivercomponent that responds to system-wide broadcast announcements e.g. screen turned off, battery low, picture captured; broadcast delivered as an Intent
Componentsee Application Component
Content Providercomponent that manages a shared set of application data; data may be held, for example, in a file, SQLite DB, or web
ContextInterface to global information about an application environment.
Dalvikvirtual machine for Android; Java source -> compile -> Dalvik executable
DexDalvik Executable format. Similar to .jar file, a .dex file is used to hold a set of class definitions, but the classes are in a format suitable for Dalvik VM rather than Java VM.
DDMSDalvik Debug Monitor Server
DialogA dialog is a small window that prompts the user to make a decision or enter additional information. A dialog does not fill the screen and is normally used for modal events that require users to take an action before they can proceed.
Fragmentsub activity, or modular section of an activity; an activity may consist of several fragments; a fragment can be re-used in other activities
Intentdescription of an operation to be performed or description of results of said operation
NDKNative Development Kit; used to invoke C/C++ from Android
OATRuntime executable file (a type of ELF - Executable and Linkable Format) produced from Dex file by dex2oat for the Android Runtime (ART)
OdexOptimized Dalvik EXecutable
Platform-toolssee \Program Files\Android\android-sdk\tools, e.g. emulators, sqlite3, sdkController
SDKSoftware Development Kit
Servicecomponent that runs in background to perform long-running operations or to perform work for remote processes; no user interface
Support Librariesto allow an app to use new features on old devices; e.g. ActionBar introduced at Android 3.0 (API 11); support-v7 is used to run on devices as early as Android 2.1 (API 7). Fragment introduced at Android 3.0 (API 11); support-v4 is used to run on devices as early as Android 1.6 (API 4).
